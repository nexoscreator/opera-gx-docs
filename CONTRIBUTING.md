# 🚀 **Contributing to Opera GX Docs**  

Thank you for considering contributing to **Opera GX Docs**! 🎉 Your contributions help improve our **documentation, extensions, and mods** for the Opera GX community.  

## 📌 **How to Contribute**  

We welcome all kinds of contributions! You can:  
- 🐞 **Report Bugs** – Found an issue? Let us know!  
- 📖 **Improve Documentation** – Fix typos, add explanations, or enhance existing guides.  
- 🎭 **Submit New Mods/Extensions** – Develop and share your Opera GX creations.  
- 💡 **Suggest Features** – Have an idea? We’d love to hear it!  

---

## 🛠 **Getting Started**  

### 1️⃣ **Fork & Clone the Repository**  
First, fork this repository and then clone it to your local machine:  

```sh
git clone https://github.com/nexoscreator/opera-gx-docs.git
cd opera-gx-docs
```

### 2️⃣ **Create a New Branch**  
Before making changes, create a new branch:  

```sh
git checkout -b feature-or-fix-name
```

### 3️⃣ **Make Your Changes**  
Modify the files you want to update or add new content.  

- For **Docs**, update files in `docs/`  
- For **Mods**, add/update files in `mods/`  
- For **Extensions**, modify files in `extensions/`  

### 4️⃣ **Commit Your Changes**  
Write a clear commit message following this format:  

```sh
git commit -m "feat(docs): Added new guide for Opera GX Extensions"
```

**Example commit message formats:**  
- `fix(mods): Fixed layout issue in NexSci-Fi mod`  
- `docs: Updated publishing guide for GX Extensions`  
- `feat(extension): Added new dark mode toggle`  

### 5️⃣ **Push Changes to Your Fork**  
```sh
git push origin feature-or-fix-name
```

### 6️⃣ **Submit a Pull Request (PR)**  
Go to the original repo on GitHub and **open a PR** from your forked branch.

## 📝 **Code & Contribution Guidelines**  

### 📖 **Documentation Guidelines**  
- Keep explanations **clear and concise**.  
- Use proper **Markdown formatting** (`# Headings`, `**Bold Text**`, `- Lists`, etc.).  
- Ensure **links are valid** and point to the correct pages.  

### 🎭 **Mod Development Guidelines**  
- Follow Opera GX’s **modding standards**.  
- Ensure assets (images, sounds, effects) **are optimized**.  
- Add **descriptive names** for mod files and resources.  

### 🔌 **Extension Development Guidelines**  
- Follow **Opera GX Extension APIs & security guidelines**.  
- Ensure **cross-browser compatibility** (where applicable).  
- Maintain **performance efficiency** to avoid lag in Opera GX.

## 🐞 **Reporting Issues**  
If you find a bug, issue, or missing information, please report it using **GitHub Issues**:  

📌 **[Open an Issue](https://github.com/NexosCreator/opera-gx-docs/issues/new)**  

**Include in your report:**  
- **A clear title** (e.g., "Bug: GX Mod crashes on startup")  
- **Steps to reproduce the issue**  
- **Expected vs. actual behavior**  
- **Screenshots (if applicable)**
